# Documentation

This folder contains project documentation, diagrams, and reports.

## Contents

- **architecture-diagram.png** - System architecture overview
- **workflow-diagram.png** - User flow diagrams
- **report.docx** - Academic project report
- **screenshots/** - Application screenshots

## Creating Documentation

### Architecture Diagram

Should include:
- Frontend (HTML/CSS/JS)
- Backend (Flask)
- Smart Contract (Solidity)
- Blockchain (Ethereum)
- MetaMask integration

### Workflow Diagram

Should show:
1. Notarization flow
2. Verification flow
3. Data flow between components

### Screenshots to Include

- Homepage
- Notarization page (before/after)
- Verification page (success/failure)
- MetaMask connection
- Transaction confirmation

## Report Structure (Suggested)

1. **Abstract**
2. **Introduction**
   - Problem statement
   - Objectives
3. **Literature Review**
   - Blockchain technology
   - Document notarization systems
4. **System Design**
   - Architecture
   - Technologies used
   - Smart contract design
5. **Implementation**
   - Backend development
   - Frontend development
   - Blockchain integration
6. **Testing & Results**
   - Test cases
   - Performance analysis
   - Gas costs
7. **Conclusion**
8. **Future Work**
9. **References**
10. **Appendices**
    - Code snippets
    - Screenshots
    - User manual
